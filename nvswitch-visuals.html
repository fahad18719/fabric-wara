Visual 01 — The Core Primitive
What a Crossbar Actually Is
Click any GPU pair to watch data route through the intersection grid. Multiple pairs can transfer simultaneously — that's the non-blocking guarantee.
NVSwitch Crossbar Matrix — Interactive
A crossbar is an N×N grid of switch points in silicon. Every row is a sender, every column is a receiver. When GPU 3 sends to GPU 7, only that intersection opens — every other pair continues unaffected. This is why NVSwitch is "non-blocking": there is no shared bottleneck. The wire between GPUs and the chip is high-speed copper on the PCB baseboard. Inside the chip, it's metal traces on silicon.

Visual 03 — Rack Scale
Inside the GB200 NVL72
72 GPUs. 9 NVSwitch tray boards. 130 TB/s all-to-all. Hover over each layer to understand what it does.
GB200 NVL72 Rack · Layer Anatomy
The NVL72 is a full rack where 36 compute trays (2 B200 GPUs each) are connected by 9 NVSwitch tray boards. Every GPU has 1.8 TB/s NVLink 5.0 bandwidth and can address any other GPU's HBM memory directly — 13.5 TB of unified memory across the rack.
Visual 04 — The Two Fabric Jobs
Scale-Up vs Scale-Out: Different Layers, Different Tools
The biggest source of confusion. NVLink/UALink live inside the pod. Ultra Ethernet lives between pods. Neither replaces the other.
Two-Fabric Architecture · Datacenter View
Scale-up fabric (NVLink/NVSwitch or UALink) uses load/store memory semantics — GPU-to-GPU feels like accessing local memory. Scale-out fabric (InfiniBand / Ultra Ethernet) uses message-passing — you send a packet, it travels through the network. These are fundamentally different communication models solving fundamentally different problems.
